# Security policy

## Supported versions

Nodus is currently pre-1.0. Security fixes are applied to the latest release
and the `main` branch.

## Reporting a vulnerability

Do not open a public issue for a suspected vulnerability. Use GitHub's private
security advisory flow for this repository:

1. Open the repository's **Security** tab.
2. Choose **Advisories** and **Report a vulnerability**.
3. Include affected versions, impact, reproduction steps, and any suggested
   mitigation.

Reports involving generated RLS, authorization, synchronization integrity,
cross-account data exposure, migration safety, or credential handling are
treated as security issues. You should receive an acknowledgement within seven
days.
